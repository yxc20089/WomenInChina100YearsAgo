# Objective status

Status at 2026-07-18. “Implemented” means exercised on a bounded three-page,
three-volume 1924–1926 slice unless the evidence column says otherwise; it does not imply
corpus-scale production readiness or historical validity.

| Objective | Status | Evidence and next gate |
|---|---|---|
| Identify available modalities | Audited | The readable `sb_raw/` prefix contains 395 PDFs, 5 DjVu files and 1 JPEG across 400 volumes/340,511 known pages (~157.4 GiB). No audio, video or born-digital text was found in that authorized prefix. |
| Select OCR/document models | Single-model first-build implemented | Pinned HunyuanOCR 1.5 is the sole learned layout/OCR component. Official `spotting_json` supplies coordinate-bearing lines and `layout_parse` supplies the structural/order proposal on the same immutable page bytes. There is no Paddle, deterministic layout authority, or OCR fallback. Historical gold accuracy and confidence calibration remain unmeasured. |

Native JSON-schema responses are whole-response validated; code derives exact offsets from surface plus context. There is no 0.8B first pass, 35B fallback, or GLiNER production path. |

| Durable multimodal ingestion | Single-model page DAG and reviewed-unit semantic runner implemented | PostgreSQL DAG, leases, retries, provenance checks, resumable workers, failure propagation, cancellation and replay are live-tested. Page-level NER and global linking are excluded from new plans. |
| Relational evidence store | E2E authority implemented | PostgreSQL stores versioned visual-model outputs, immutable text versions and alignments, exact evidence spans, reviewed text selections, coherent-unit semantic inputs, mention occurrences, local clusters, events, participants, evidence, confidence calibration records, and job control. | Search/retrieval store | Implemented baseline | Active-selection OpenSearch CJK/BGE-M3/RRF contains 2,498 regions across 1924–1926 and returns exact derivative/region pointers. Three year-specific live queries passed; historian questions, reranking and larger-corpus evaluation remain. |

The current graph correctly remains empty because no genuine evidence has passed all review gates. |

| Researcher UI | Implemented bounded workflow | Natural-language lexical/dense/hybrid search, scan links, segmentation JSON edit/review/activation, mention/entity/claim review, reviewed insights and evidence-linked machine exploration are available locally. | LLM research assistant | Bounded handoff and executable quality-evaluation boundary implemented | Each question performs fresh retrieval; at most 12 prior turns are untrusted continuity context. Local OpenAI-compatible transport is live-fixture tested; remote use requires HTTPS and explicit data-egress consent, redirects are blocked, and model/revision/configuration plus prompt/context/output hashes, provider token usage and contract-based cost are exposed. Missing/foreign citations reject and withhold model text. A frozen benchmark runner now enforces historian-authored case eligibility, expected-region scoring, model-blind two-review/adjudication and paired case bootstrap comparison. No approved live model, eligible generation set, human scores or winner exists. |
| Scene/lived-experience reconstruction | Structural safety and evaluation gates implemented; content intentionally blocked | Scene generation hard-abstains without directly reviewed claims. A returned scene must order Direct evidence, Plausible reconstruction and Speculative details and cite a directly reviewed claim inside Direct evidence; otherwise its raw text is withheld. Scene cases are included in the frozen generation protocol, but historical review and model-quality evaluation must still precede any reconstructed narrative. |
| New historical insights | Not yet established | The exploratory panel surfaces scan-linked reading priorities and model disagreement, while the reviewed insight plane remains empty. Only historian-reviewed evidence can support a historical finding. |

The next scientific bottleneck is not another unscored model run. It is a
representative, issue-split, double-reviewed OCR/layout/NER benchmark, followed
by an independently adjudicated relation set over those exact mentions, and a historian-authored
retrieval set. The next product bottleneck is configuring and evaluating the
citation-validating assistant over a historian-authored frozen generation set,
then adding controlled conversation persistence only if researcher studies
justify it. The next implementation exercise is one historian-approved Holbein
coherent unit through `wic-e2e`, mention/identity/event review, and a non-empty
forward/reverse graph proof; the code refuses to manufacture that approval.
